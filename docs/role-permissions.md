# 角色与权限说明

本系统当前保留四类角色。数据库和后端接口统一使用英文角色编码，前端界面统一显示中文角色名。

| 后端编码 | 前端显示 | 作用定位 | 主要权限 |
| :--- | :--- | :--- | :--- |
| `ADMIN` | 系统管理员 | 全权限角色，负责系统演示、数据模拟、账号和权限统筹。 | 拥有其他所有角色权限，可查看采样、告警、工单和历史数据，可启动/停止模拟测试并切换异常数据。 |
| `OPERATOR` | 运行监控员 | 面向值班监控，关注实时运行、采样数据和告警。 | 可查询采样监测数据、告警记录和历史数据；不可查看工单详情，不可处理工单，不可使用模拟测试。 |
| `ENGINEER` | 维保工程师 | 面向现场处理，关注告警和维保工单闭环。 | 可查询告警、工单，以及工单/告警关联的采样数据；可处理未分派或自己相关工单；不可启动模拟测试。 |
| `MANAGER` | 管理人员 | 面向管理分析，关注设备运行、告警和工单处理状态。 | 可只读查询采样、告警、工单和历史数据；不可处理工单，不可使用模拟测试。 |

## 权限原则

- `ADMIN` 是全权限者，后端权限判断中总是通过。
- 前端负责隐藏无权限菜单和表单项，后端仍必须按角色过滤和拦截。
- 注册、登录、数据库约束和接口请求都只使用英文编码：`ADMIN`、`OPERATOR`、`ENGINEER`、`MANAGER`。
- 中文名称只用于界面展示，不写入 `SYS_USER.ROLE_CODE`。

## 查询内容

“数据库消息查询”统一覆盖三类业务数据：

- 采样监测数据：来自 `TS_RAW_DATA`，展示设备、测点、采样时间、数值、采样频率和质量。
- 告警记录：来自 `ALARM_LOG`，展示设备、测点、告警类型、等级、开始/结束时间和状态。
- 维保工单：来自 `MAINT_TASK`，关联 `ALARM_LOG` 展示工单状态、负责人、反馈和告警来源。

## 后续调整

角色体系目前不增删，便于匹配现有数据库约束和种子账号。后续如果需要新增或合并角色，必须同步修改：

- `SYS_USER` 的 `CK_SYS_USER_ROLE` 约束。
- 基础账号初始化脚本。
- 后端角色枚举/权限判断。
- 前端 `RoleCode` 类型和中文展示映射。
- 本文档和 agent 记忆文档。

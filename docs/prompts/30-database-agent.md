# 数据库开发 Agent 提示词

你负责 Oracle 21c XE 数据库脚本、索引、序列、基础数据和查询兼容性。所有 SQL 必须能在 Oracle 21c XE 上执行。

## 关键约束

- 当前项目主键自增使用显式 `SEQUENCE`。
- 查询分页可以使用 21c 支持的语法，但要优先匹配现有 MyBatis 实现风格。
- 字符串使用 `VARCHAR2`。
- 数值使用 `NUMBER`。
- 高频采样时间建议使用 `TIMESTAMP`。
- 所有外键约束命名清晰，便于排错。

## 建议脚本位置

```text
Core/src/main/resources/db/oracle21c-init.sql
Core/src/main/resources/db/oracle21c-schema.sql
Core/src/main/resources/db/oracle21c-base-data.sql
Core/src/main/resources/db/oracle21c-mock-data.sql
```

`oracle21c-init.sql` 只作为入口脚本，按顺序调用 schema、base-data、mock-data。部署生产或课程演示环境时，结构框架、部署基础数据和模拟运行数据必须分开维护。

## 核心表

### STATION_BASE

字段建议：

- `ID NUMBER(19) PRIMARY KEY`
- `NAME VARCHAR2(100) NOT NULL`
- `VOLTAGE_LEVEL VARCHAR2(20)`
- `LOCATION VARCHAR2(200)`
- `CREATED_AT TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL`

### BAY_BASE

字段建议：

- `ID NUMBER(19) PRIMARY KEY`
- `STATION_ID NUMBER(19) NOT NULL`
- `NAME VARCHAR2(100) NOT NULL`
- `BAY_TYPE VARCHAR2(50)`

### DEVICE_BASE

字段建议：

- `ID NUMBER(19) PRIMARY KEY`
- `BAY_ID NUMBER(19) NOT NULL`
- `NAME VARCHAR2(100) NOT NULL`
- `DEVICE_TYPE VARCHAR2(50) NOT NULL`
- `CURRENT_LIMIT NUMBER(12, 3)`
- `TEMP_LIMIT NUMBER(12, 3)`
- `TEMP_RATE_LIMIT NUMBER(12, 3)`
- `STATUS NUMBER(2) DEFAULT 0 NOT NULL`

### TAG_BASE

字段建议：

- `ID NUMBER(19) PRIMARY KEY`
- `DEVICE_ID NUMBER(19) NOT NULL`
- `TAG_CODE VARCHAR2(64) NOT NULL`
- `TAG_NAME VARCHAR2(100) NOT NULL`
- `UNIT VARCHAR2(20)`
- `WARN_LIMIT NUMBER(12, 3)`
- `RATE_LIMIT NUMBER(12, 3)`

### TS_RAW_DATA

字段建议：

- `ID NUMBER(19) PRIMARY KEY`
- `DEVICE_ID NUMBER(19) NOT NULL`
- `TAG_ID NUMBER(19)`
- `SAMPLE_TIME TIMESTAMP NOT NULL`
- `VAL NUMBER(14, 4) NOT NULL`
- `FREQ_FLAG NUMBER(1) NOT NULL`
- `QUALITY_FLAG NUMBER(2) DEFAULT 0 NOT NULL`
- `CREATED_AT TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL`

`FREQ_FLAG`：

- `0`：1 分钟常规采样。
- `1`：1 秒高频采样。

### ALARM_LOG

字段建议：

- `ID NUMBER(19) PRIMARY KEY`
- `DEVICE_ID NUMBER(19) NOT NULL`
- `TAG_ID NUMBER(19)`
- `ALARM_TYPE VARCHAR2(50) NOT NULL`
- `ALARM_LEVEL VARCHAR2(20) NOT NULL`
- `START_TIME TIMESTAMP NOT NULL`
- `END_TIME TIMESTAMP`
- `START_VAL NUMBER(14, 4)`
- `END_VAL NUMBER(14, 4)`
- `STATUS NUMBER(2) DEFAULT 0 NOT NULL`

### MAINT_TASK

字段建议：

- `TASK_ID NUMBER(19) PRIMARY KEY`
- `ALARM_ID NUMBER(19) NOT NULL`
- `STATUS NUMBER(2) DEFAULT 0 NOT NULL`
- `ASSIGNEE VARCHAR2(100)`
- `FEEDBACK VARCHAR2(1000)`
- `CREATED_AT TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL`
- `UPDATED_AT TIMESTAMP`
- `FINISHED_AT TIMESTAMP`

`STATUS`：

- `0`：待办。
- `1`：处理中。
- `2`：已完成。

## 序列要求

每张核心表独立序列：

```sql
CREATE SEQUENCE SEQ_STATION_BASE START WITH 1 INCREMENT BY 1 NOCACHE;
CREATE SEQUENCE SEQ_BAY_BASE START WITH 1 INCREMENT BY 1 NOCACHE;
CREATE SEQUENCE SEQ_DEVICE_BASE START WITH 1 INCREMENT BY 1 NOCACHE;
CREATE SEQUENCE SEQ_TAG_BASE START WITH 1 INCREMENT BY 1 NOCACHE;
CREATE SEQUENCE SEQ_TS_RAW_DATA START WITH 1 INCREMENT BY 1 CACHE 1000;
CREATE SEQUENCE SEQ_ALARM_LOG START WITH 1 INCREMENT BY 1 NOCACHE;
CREATE SEQUENCE SEQ_MAINT_TASK START WITH 1 INCREMENT BY 1 NOCACHE;
```

高频表 `SEQ_TS_RAW_DATA` 可以使用 `CACHE 1000`，减少序列争用。

## 索引要求

必须建立：

```sql
CREATE INDEX IDX_TS_RAW_DEVICE_TIME ON TS_RAW_DATA (DEVICE_ID, SAMPLE_TIME);
CREATE INDEX IDX_TS_RAW_TAG_TIME ON TS_RAW_DATA (TAG_ID, SAMPLE_TIME);
CREATE INDEX IDX_ALARM_DEVICE_TIME ON ALARM_LOG (DEVICE_ID, START_TIME);
CREATE INDEX IDX_TASK_ALARM ON MAINT_TASK (ALARM_ID);
```

## 分区策略

`TS_RAW_DATA` 是最大表。建议提供两种脚本：

- 非分区版：兼容所有 Oracle XE 环境，适合课程设计和本地演示。
- 分区版：按月 Range Partition，适合说明优化策略。

分区版示例：

```sql
CREATE TABLE TS_RAW_DATA (
  ID NUMBER(19) NOT NULL,
  DEVICE_ID NUMBER(19) NOT NULL,
  TAG_ID NUMBER(19),
  SAMPLE_TIME TIMESTAMP NOT NULL,
  VAL NUMBER(14, 4) NOT NULL,
  FREQ_FLAG NUMBER(1) NOT NULL,
  QUALITY_FLAG NUMBER(2) DEFAULT 0 NOT NULL,
  CREATED_AT TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,
  CONSTRAINT PK_TS_RAW_DATA PRIMARY KEY (ID)
)
PARTITION BY RANGE (SAMPLE_TIME) (
  PARTITION P_TS_202605 VALUES LESS THAN (TIMESTAMP '2026-06-01 00:00:00'),
  PARTITION P_TS_202606 VALUES LESS THAN (TIMESTAMP '2026-07-01 00:00:00'),
  PARTITION P_TS_MAX VALUES LESS THAN (MAXVALUE)
);
```

如果本地 Oracle XE 不支持分区能力，应使用非分区版，并在文档中说明生产环境优化方案。

## 查询要求

分钟级趋势：

```sql
SELECT DEVICE_ID, TAG_ID, SAMPLE_TIME, VAL, FREQ_FLAG
FROM TS_RAW_DATA
WHERE DEVICE_ID = #{deviceId}
  AND SAMPLE_TIME BETWEEN #{startTime} AND #{endTime}
  AND FREQ_FLAG = 0
ORDER BY SAMPLE_TIME
```

告警追忆：

```sql
SELECT DEVICE_ID, TAG_ID, SAMPLE_TIME, VAL, FREQ_FLAG
FROM TS_RAW_DATA
WHERE DEVICE_ID = #{deviceId}
  AND SAMPLE_TIME BETWEEN #{startTime} AND #{endTime}
ORDER BY SAMPLE_TIME
```

Oracle 21c 支持 `OFFSET/FETCH`，但现阶段后端如未封装分页组件，可继续使用 `ROWNUM` 包装保持兼容风格。

## 验收标准

- 建表脚本可重复说明执行顺序。
- 表、序列、索引、外键齐全。
- `TS_RAW_DATA` 具备 `FREQ_FLAG`。
- 查询能按设备和时间范围走复合索引。
- SQL 兼容 Oracle 21c XE。
